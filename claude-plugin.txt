⏺ 조사 결과입니다. 템플릿 치환 변수와 실제 OS 환경 변수를 구분하는 것이
  핵심입니다.

  ---
  Skill 스크립트에서 접근 가능한 환경 변수

  실제 OS-level 환경 변수 (스크립트가 process.env / os.environ으로 읽을 수 있는 
  것)

  자동 주입되는 환경 변수는 단 하나입니다:

  ┌────────────┬─────┬───────────────────────────────────────┐
  │    변수    │ 값  │                 설명                  │
  ├────────────┼─────┼───────────────────────────────────────┤
  │ CLAUDECODE │ 1   │ Claude Code가 생성한 셸 환경임을 표시 │
  └────────────┴─────┴───────────────────────────────────────┘

  그 외 CLAUDE_SESSION_ID, CLAUDE_SKILL_DIR, CLAUDE_PLUGIN_ROOT 등은 스크립트의 
  환경 변수로 주입되지 않습니다.

  ---
  템플릿 치환 변수 (SKILL.md 마크다운에서 치환되는 것)

  이들은 Claude가 콘텐츠를 보기 전에 문자열 치환됩니다. 스크립트 경로를 조립하는
   데는 쓸 수 있지만, 스크립트 내부에서 환경 변수로 접근할 수는 없습니다.

  ┌───────────────────────┬──────────────────────────────────────┐
  │         변수          │                 설명                 │
  ├───────────────────────┼──────────────────────────────────────┤
  │ ${CLAUDE_SESSION_ID}  │ 현재 세션 ID                         │
  ├───────────────────────┼──────────────────────────────────────┤
  │ ${CLAUDE_SKILL_DIR}   │ SKILL.md가 위치한 디렉토리 절대 경로 │
  ├───────────────────────┼──────────────────────────────────────┤
  │ ${CLAUDE_PLUGIN_ROOT} │ 플러그인 설치 디렉토리               │
  ├───────────────────────┼──────────────────────────────────────┤
  │ ${CLAUDE_PLUGIN_DATA} │ 플러그인 영속 데이터 디렉토리        │
  ├───────────────────────┼──────────────────────────────────────┤
  │ $ARGUMENTS, $0, $1... │ skill 호출 인자                      │
  └───────────────────────┴──────────────────────────────────────┘

  예시: SKILL.md에 !`${CLAUDE_SKILL_DIR}/run.sh`라고 쓰면, 실행 시점에
  /Users/.../my-skill/run.sh로 치환되어 실행됩니다. 하지만 run.sh 내부에서
  $CLAUDE_SKILL_DIR 환경 변수를 읽을 수는 없습니다.

  ---
  스크립트에 값을 전달하는 방법

  스크립트에서 이 값들이 필요하면:

  1. 인자로 전달: !`${CLAUDE_SKILL_DIR}/run.sh ${CLAUDE_SESSION_ID}`
  2. SessionStart hook으로 환경 변수 영속화:
  # SessionStart hook script
  if [ -n "$CLAUDE_ENV_FILE" ]; then
    echo 'export MY_VAR=value' >> "$CLAUDE_ENV_FILE"
  fi

  ---
  컨텍스트별 요약

  ┌─────────────────────┬────────────┬────────────────────┬─────────────┐
  │      컨텍스트       │ CLAUDECODE │  $CLAUDE_ENV_FILE  │ 템플릿 변수 │
  ├─────────────────────┼────────────┼────────────────────┼─────────────┤
  │ Skill 스크립트 실행 │     O      │         X          │ O (치환됨)  │
  ├─────────────────────┼────────────┼────────────────────┼─────────────┤
  │ Hook 스크립트       │ X (의도적) │ O (SessionStart만) │      X      │
  ├─────────────────────┼────────────┼────────────────────┼─────────────┤
  │ MCP stdio 서버      │     O      │         X          │ O (설정 내) │
  └─────────────────────┴────────────┴────────────────────┴─────────────┘

  핵심: 템플릿 변수는 컴파일 타임(마크다운 전처리), CLAUDECODE=1은
  런타임(서브프로세스 환경)입니다.

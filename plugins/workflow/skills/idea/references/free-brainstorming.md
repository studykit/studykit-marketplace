# Free Brainstorming

## What It Is
Classic brainstorming where participants freely generate as many ideas as possible without judgment or filtering. Quantity is the goal — evaluation comes later.

## When to Use
- Starting from a blank slate with no existing solution
- Early-stage exploration of a broad topic
- The team needs to warm up before deeper analysis

## When NOT to Use
- Ideas are already abundant and need organizing
- A specific existing solution needs systematic improvement (use SCAMPER instead)
- Decision-making is the immediate goal

## Process Steps
1. **Set the topic clearly** — Restate the brainstorming question in a specific, actionable form. ("How might we..." format works well.)
2. **Rapid-fire generation** — Generate ideas as fast as possible. Write each one down. No evaluation, no discussion, no "but..."
3. **Build on others' ideas** — Take any idea and extend it, combine it, or flip it. Use "Yes, and..." to chain ideas together.
4. **Push past the obvious** — After the first wave (usually 5-8 ideas), push further. The obvious ideas come first; the creative ones come after the mental "wall."
5. **Wild ideas welcome** — Encourage impractical or absurd ideas. They often contain seeds of practical innovation.
6. **Capture everything** — Number each idea for easy reference. Don't wordsmith — a rough phrase is fine.

## Facilitator Prompts
- "What's the first thing that comes to mind?"
- "What else? Keep going — don't filter."
- "That's interesting — what if we pushed that idea further?"
- "What would a completely different industry do with this problem?"
- "What's the most ridiculous solution you can think of?"
- "We have [N] ideas so far. Let's aim for [N+5] more before we pause."
- "What if money/time/technology were no constraint?"

## Example
**Topic**: How might we improve the onboarding experience for new developers?

1. Interactive tutorial that builds a real mini-project
2. Buddy system — pair each new dev with a mentor for the first week
3. Video walkthrough of the architecture
4. "First PR in 30 minutes" challenge
5. Automated environment setup script
6. Onboarding scavenger hunt through the codebase
7. Weekly "ask anything" sessions with senior engineers
8. Self-paced learning path with checkpoints
9. Shadowing rotation across teams
10. Gamified progress tracker with badges

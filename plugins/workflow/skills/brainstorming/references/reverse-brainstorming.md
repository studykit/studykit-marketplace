# Reverse Brainstorming

## What It Is
Instead of asking "How do we solve this?", ask "How could we make this problem worse?" or "How could we guarantee failure?" Then reverse those answers into solutions. This paradoxical approach bypasses mental blocks and reveals hidden assumptions.

## When to Use
- The group is stuck and can't generate new ideas
- The problem feels too familiar — people default to the same solutions
- Need to identify failure modes or risks
- Want to challenge assumptions about what "good" looks like

## When NOT to Use
- Ideas are already flowing well (don't fix what isn't broken)
- The topic is sensitive and "making it worse" could feel uncomfortable
- Already in convergent phase — need to evaluate, not generate

## Process Steps
1. **State the problem clearly** — Write down the original goal or question.
2. **Reverse the question** — Flip it: "How could we make [problem] as bad as possible?" or "How could we guarantee [goal] fails?"
3. **Brainstorm the worst ideas** — Generate ideas enthusiastically. The worse, the better. This is usually the fun part — people loosen up.
4. **Reverse each bad idea** — Take each "worst idea" and flip it into a positive solution. "Ignore all user feedback" → "Build systematic feedback loops at every touchpoint."
5. **Identify hidden insights** — Some reversed ideas will be obvious. Others will reveal assumptions or blind spots you hadn't considered.
6. **Select the strongest reversals** — Pick the solutions that feel genuinely novel or that address gaps in your current thinking.

## Facilitator Prompts
- "Forget about solving this. How could we make this problem absolutely terrible?"
- "What's the fastest way to guarantee this project fails?"
- "What would the worst possible version of this look like?"
- "Great worst idea! Now flip it — what's the opposite?"
- "This reversal is interesting — it reveals an assumption we've been making. What is it?"
- "Which of these reversed solutions surprises you? Those are usually the gold."

## Example
**Problem**: How to improve customer support response time?

**Worst ideas** → **Reversed solutions**:
| Make It Worse | Reversed Solution |
|---------------|-------------------|
| Hide the contact button | Make support accessible from every page with one click |
| Route tickets randomly | Smart routing based on issue type and agent expertise |
| Never train agents | Continuous training program with weekly knowledge sessions |
| Delete all documentation | Build searchable, AI-powered knowledge base for self-service |
| Respond only on Fridays | Implement real-time chat with AI triage for instant first response |
| Punish agents for fast responses | Incentivize and celebrate quick resolution times |

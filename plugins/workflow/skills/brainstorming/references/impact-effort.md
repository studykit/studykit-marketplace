# Impact-Effort Matrix

## What It Is
A 2x2 prioritization framework that plots ideas on two axes — **Impact** (how much value it delivers) and **Effort** (how much work it requires). Creates four quadrants that make prioritization visual and intuitive.

## When to Use
- Have multiple ideas or options and need to prioritize
- Resources (time, money, people) are limited
- Need a quick, intuitive way to compare options
- Transitioning from brainstorming to action planning

## When NOT to Use
- Divergent phase — still generating ideas
- Only 1-2 options to compare (use Pros & Cons instead)
- Decision requires nuanced multi-criteria analysis beyond just impact and effort

## Process Steps
1. **List all options** — Gather all ideas, features, or initiatives to evaluate.
2. **Define "Impact"** — What does impact mean for this context? Revenue? User satisfaction? Risk reduction? Strategic value? Agree on the definition.
3. **Define "Effort"** — What does effort include? Development time? Cost? Complexity? Organizational change required?
4. **Plot each item** — For each option, estimate its impact (High/Low) and effort (High/Low). Place it in the appropriate quadrant.
5. **Discuss placements** — Challenge assumptions. "Is this really high impact?" "Could we reduce the effort by scoping it differently?"
6. **Prioritize by quadrant**:
   - **Quick Wins** (High Impact, Low Effort) → Do these first
   - **Major Projects** (High Impact, High Effort) → Plan and schedule these
   - **Fill-ins** (Low Impact, Low Effort) → Do when you have spare capacity
   - **Avoid** (Low Impact, High Effort) → Deprioritize or drop these

## Facilitator Prompts
- "On a scale of impact, how much value would [idea] deliver if we did it well?"
- "How much effort — time, resources, complexity — would [idea] require?"
- "We've placed [idea] in Quick Wins. Does everyone agree, or should we challenge that?"
- "Are there any items in 'Avoid' that we could rescope to reduce effort and move to Quick Wins?"
- "What's our top Quick Win? Let's start there."
- "For Major Projects, which one should we plan for next quarter?"

## Example
**Context**: Prioritizing features for a SaaS product

```
                        High Impact
                            │
         Major Projects     │     Quick Wins
         ─────────────      │     ──────────
    • API redesign          │  • Search autocomplete
    • Multi-language support │  • Bulk export feature
    • Mobile app            │  • Onboarding tutorial
                            │  • Dark mode
High Effort ────────────────┼──────────────── Low Effort
                            │
         Avoid              │     Fill-ins
         ─────              │     ────────
    • Custom reporting      │  • Footer redesign
      engine                │  • Email template update
    • Blockchain            │  • Admin page styling
      integration           │
                            │
                        Low Impact
```

**Action plan**:
1. Start with Quick Wins: Search autocomplete → Bulk export → Onboarding tutorial
2. Plan Major Project: API redesign (schedule for Q2)
3. Fill-ins: Pick up during light sprint weeks
4. Avoid: Remove blockchain integration from backlog; revisit custom reporting only if customer demand increases

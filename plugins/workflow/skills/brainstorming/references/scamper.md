# SCAMPER

## What It Is
A structured creativity technique that uses seven action verbs to systematically transform an existing product, process, or idea: **S**ubstitute, **C**ombine, **A**dapt, **M**odify, **P**ut to another use, **E**liminate, **R**everse.

## When to Use
- Improving or innovating on something that already exists
- Feeling stuck with an existing product or process
- Need systematic exploration rather than random brainstorming

## When NOT to Use
- Starting from a completely blank slate with nothing to transform
- The problem requires root cause analysis, not idea generation
- Convergent phase — need to evaluate, not generate

## Process Steps
1. **Define the target** — Clearly state what existing thing you're improving (product, service, process, feature).
2. **Walk through each SCAMPER lens**, spending 2-3 minutes on each:
   - **Substitute**: What components, materials, people, or processes can be replaced?
   - **Combine**: What can be merged? Features, functions, steps, markets?
   - **Adapt**: What else is like this? What ideas can be borrowed from other domains?
   - **Modify** (Magnify/Minify): What can be made bigger, smaller, faster, slower, more frequent, less frequent?
   - **Put to another use**: How can this be used in a different context or by a different audience?
   - **Eliminate**: What can be removed, simplified, or reduced?
   - **Reverse** (Rearrange): What if the order were flipped? Roles swapped? Cause and effect reversed?
3. **Capture ideas per lens** — Don't skip lenses even if they seem unproductive at first.
4. **Cross-pollinate** — After all lenses, look for combinations across lenses.

## Facilitator Prompts
- **Substitute**: "What if you replaced [component] with something completely different?"
- **Combine**: "What if you merged [feature A] and [feature B] into one?"
- **Adapt**: "How does [other industry] solve a similar problem?"
- **Modify**: "What if this were 10x bigger? 10x smaller? 10x faster?"
- **Put to another use**: "Who else could benefit from this, besides the current users?"
- **Eliminate**: "What would happen if you removed [step/feature] entirely?"
- **Reverse**: "What if the user did [step B] before [step A]?"

## Example
**Target**: A weekly team standup meeting

- **Substitute**: Replace synchronous meeting with async video updates
- **Combine**: Merge standup with sprint planning into one session
- **Adapt**: Borrow the "walking meeting" format from design thinking
- **Modify**: Shorten from 15 minutes to 5 minutes; increase from weekly to daily
- **Put to another use**: Use standup data to auto-generate weekly progress reports
- **Eliminate**: Remove status updates — focus only on blockers
- **Reverse**: Start with blockers first instead of progress; let the team lead last instead of first

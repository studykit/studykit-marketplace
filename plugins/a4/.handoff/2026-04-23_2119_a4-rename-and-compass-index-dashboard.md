## Decisions locked in this session

These are the session's authoritative outcomes. When in doubt the next session should follow them rather than reopen.

### Path convention

- The workspace root is `a4/` (lowercase) ‚Äî not `A4/`. All skill docs under `plugins/think/skills/` have been sed-replaced. The plugin's own stray fixture at `plugins/think/A4/` was renamed to `plugins/think/a4/` via a two-step `git mv` (APFS is case-insensitive, so the direct `A4 ‚Üí a4` rename is rejected).
- Historical references to `A4/` inside `plugins/think/a4/2026-04-12-think-pipeline-restructuring.decide.md` point to *other repos' directories at a past time* and were intentionally not touched.

### INDEX dashboard

- `a4/INDEX.md` is a **view**, not a primary artifact. The per-stage files and their frontmatter are the source of truth.
- It is **regenerated on every `compass` invocation**. Not session-closing hooks, not per-skill ‚Äî only compass.
- It is **committed to git**, but compass does **not** auto-commit. The file is simply left in the working tree; the next session-closing (of any skill) or the user sweeps it up.
- Row granularity: one row per topic, stages as columns (`Usecase | Arch | Bootstrap | Plan | Spec`).
- Cell vocabulary is exactly five icons, with a single-rule priority ladder `‚úó > ! > ‚ü≥ > ‚úì > ‚Äî`:
  - `‚úó` ‚Äî `status: blocked`
  - `!` ‚Äî unreflected derivative files exist (reviews/research/exploration/test-report/integration-report not listed in `reflected_files`)
  - `‚ü≥` ‚Äî `status: draft / verified / implementing / in-review / pass / fail`
  - `‚úì` ‚Äî `status: complete / final`
  - `‚Äî` ‚Äî stage file does not exist
- **There is no "Next action" column and none is planned.** INDEX is a Í≤ΩÎ≥¥Ìåê (alert board); the per-topic narrative stays in compass's Step 3 on-demand diagnosis. This was the outcome of a deliberate simplification after an earlier over-engineered proposal with seven derivation rules.
- Reading cost: stage-file frontmatter only (~30 lines each) + a glob of `a4/<topic>.*.md` for file names. Derivative file bodies are **never read** for INDEX generation. The frontmatter `reflected_files` list was designed to make this possible.

### Spark / pipeline relationship ‚Äî model M3 (loose coupling)

- Parallel tools by default. Not a funnel.
- A spark file *may* opt into a lifecycle via optional frontmatter: `status: open | promoted | discarded`, `graduated_to: <slug>`, `discarded_reason: <text>`. Missing = treated as `open`.
- A pipeline `usecase.md` may optionally record provenance with `origin: spark/<filename>`. One-way link (spark ‚Üí pipeline direction) is sufficient; bidirectional is not enforced.
- Compass's Step 0 **reads these fields when present** ‚Äî implementation landed on the read side only. The **write side** (having spark-brainstorm and spark-decide actually set `status: promoted` when the user graduates a spark, etc.) is still pending.

### Archive model

- Archive = **folder location** (`a4/archive/`). No `archived: true` frontmatter field.
- Move is always user-confirmed. Compass offers the move only when a topic reaches `status: complete` or `status: final`.
- Archived topics are listed in a separate INDEX section, excluded from the Pipeline table.
- An earlier proposal to introduce an `archived: true` frontmatter field was **withdrawn**. Do not re-add it.

### Coverage ‚Äî what INDEX covers right now vs. later

- **This round**: Pipeline table (+ Spark section when any open sparks exist, + Archive section when `a4/archive/` has content). Sections are dynamic ‚Äî absent when the corresponding artifacts are absent.
- **Deferred** (explicitly parked, not dropped):
  - `a4/inbox.md` ‚Äî one-line pre-mature ideas with `active | promoted | discarded` lifecycle. Callgraph-service pattern as reference.
  - Experiments / PoC slot ‚Äî naming and location to be decided.
  - `DECISIONS.md` + `decisions/D##-*.md` (callgraph-service style ADR set) ‚Äî whether to introduce and how it relates to spark-decide output.
